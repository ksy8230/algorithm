객체지향 특징 : 객체 지향을 요리의 레시피에 빗댄다면 요리의 정보들을 담고 있는 레시피가 'constructor'가 되고 레시피가 바뀌면 레시피를 바라보고 만들어진 요리들이 모두 바뀌는 것처럼 객체들도 한 번에 수정이 가능합니다. 이 때문에 코드의 중복 생성을 줄일 수 있고 재사용 또한 높아집니다. 단점 => 프로토타입으로 보완 : 생성자(constructor) 안에 있는 프로퍼티들은 객체를 만들 때마다 생성이 됩니다. 즉, 많은 객체를 생산할 수록 메모리가 많이 사용된다는 의미입니다. 이런 문제를 해결하기 위해 프로토타입을 사용합니다. 프로토타입은 모든 객체에 공통으로 사용할수도 있고 필요할 때 재정의가 되기 때문에 커스터마이징도 가능합니다. 컨스트럭터의 역할 : 객체를 만든다 / 내부에 초기 프로퍼티를 만든다 객체와 객체를 상속할 때 __proto__를 사용하고 / 다른 방법으로 Object.create(객체) 를 사용해 상속 받기도 한다 모든 함수는 call을 갖고 있다. call은 상속과 super를 사용하지 않고 함수를 독립적으로 빼서 객체에게 사용할 때 call을 사용한다.
